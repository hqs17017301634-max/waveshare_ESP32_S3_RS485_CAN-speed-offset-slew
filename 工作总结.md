## 4. CAN B 灯光/拨杆 关键报文与字段（实车抓包结论）

- **0x249 SCCMLeftStalk**：`byte1` 低 4 位=counter(0-15 滚动)，bits4-5=status；
  status=1=PULL(超车灯/Flash-to-pass)，0=idle。`byte0 = BASE[counter] XOR OFFSET[status]`（已破解）。
  → 用于**高光爆闪**（双拉触发，8 次）。
- **0x3C2 VCLEFT_switchStatus（多路复用！）**：`byte0 bits0..1`=mux 索引。
  - **mux0**(`b0&3==0`)：承载 `byte0 bit3`=hazardButtonPressed(按下=0x08) + 计数器/CRC；此 mux 的 `data[3]=0x55` 是填充，**不是滚轮**。抓包按双闪 = `08 55 55 55 00 00 59 45`。
  - **mux1**(`b0&3==1`)：承载 `data[3]`=rightScrollTicks(6-bit 有符号，空闲 0，+前/-后)。空闲样例 `41 85 0 0 0 0 0 128`。
  - ⚠️ **坑（cap64 实测）**：不分 mux 直接读 data[3] 会把 mux0 的 0x55 误读成 +21 假滚轮，踩刹车换 R 时触发“D 意图→取消”，导致倒车双闪只闪 1~2 下就停。**已修**：滚轮只在 mux1 读；双闪注入只在 mux0（缓存最近 mux0 帧）。
  → 用于**双闪注入(mux0)** + **右滚轮+刹车 档位识别(mux1)**。
- **0x3F5 VCFRONT_lighting**：只读灯态反馈，**不注入**。
  （byte3 bit5 远光实点亮 / bit4 自动远光 / byte4 bit5 手动远光 / byte1 bit7 大灯手动模式）
- **0x273 body lighting**：后雾灯，复用最新实时帧只切 `data[2] bit7`（0x80）。
- **0x082 UI_tripPlanning**：电池预热。ON=`AF 50 AC 3C FF 03 9A 0F`，OFF=`01 50 AC 3C FF 03 9A 0F`，周期 1000ms。
- **0x109 SBW_RQ_SCCM**：换挡请求帧，**CRC/档位值未知，禁止伪造注入**（风险高）。需要再抓右拨杆 D/R 才能破解。
